# Privacy Policy

## 文档目的

本文件定义 Digital Avatar 在 MVP 范围内对用户数据处理的基本承诺，并为后续实现阶段提供隐私边界基线。

## 我们处理的数据类型

按照当前设计，系统可能处理以下数据：

- 账户与认证信息
- Avatar 基础资料
- Persona 画像摘要
- Agent 配置
- Task 输入与结果摘要
- Memory 记录
- Audit 日志
- 安全与诊断日志

## 数据处理原则

- 最小必要：只处理支撑当前能力所需的数据
- 用户可控：用户可查看、确认、拒绝、删除相关数据
- 默认审慎：高敏感信息不默认自动进入长期记忆
- 可追溯：关键操作需有审计链路
- 本地优先：优先支持本地部署与本地数据控制

## 用户权利

MVP 至少需要支持：

- 用户数据导出
- 用户数据删除
- 数据保留期限说明
- 敏感数据分类与分级说明

这些能力的设计要求见：

- `docs/02-architecture/security-architecture.md`
- `docs/01-product/non-functional-requirements.md`
- `docs/05-review/security-checklist.md`

## 数据保留

当前方案要求建立明确的数据保留策略，但代码实现尚未落库。后续实现时，必须为以下数据定义保留规则：

- 账户数据
- Task 执行记录
- Memory 记录
- Audit 日志
- 诊断日志

## 敏感数据处理

当前冻结要求：

- 敏感字段支持应用层加密能力
- 数据库文件加密作为部署选项
- 审计默认记录摘要，不记录完整敏感内容
- 前端展示与日志输出应做必要脱敏

## 第三方模型与服务

MVP 方案允许接入模型 Provider。实际部署时，用户应知悉：

- 输入给模型的内容可能离开本地进程
- 是否出网、发送何种内容、是否启用本地模型，应作为显式配置项
- 与第三方服务的具体数据边界，需在后续实现阶段补充到正式隐私文档中

## 文档状态

本文件是当前设计阶段的隐私基线，不替代未来面向最终用户发布的法律文本。进入可部署版本前，必须将本文件升级为正式隐私政策。
