# 非功能需求

## 1. 安全
- JWT + Refresh Token
- bcrypt 密码哈希
- 默认拒绝、显式授权、最小权限
- 高风险操作门禁
- LLM 输出安全过滤
- 敏感记忆默认不自动入长期库
- 用户导出、删除、保留期限机制

## 2. 性能
- 首屏核心页面加载尽可能 < 2.5s
- Task 状态查询 P95 < 500ms（不含模型执行）
- Memory pending 查询 P95 < 300ms
- Audit 列表必须分页

## 3. 可观测性
- `/health` 健康检查
- `/metrics` 基础指标
- JSON 结构化日志
- trace_id 全链路透传
- 日志与审计分离但可关联

## 4. 可维护性
- 控制器不承载核心业务逻辑
- 模块边界按领域划分
- 冻结契约优先于并行开发
- ADR 记录关键技术决策

## 5. 隐私与合规
- 数据最小化
- 用户可查看、确认、拒绝、删除相关数据
- 敏感字段支持应用层加密
- 部署侧可选数据库文件加密
- 审计默认记录摘要，不记录完整敏感内容

## 6. 可用性
- 引导式首启
- 统一加载态、错误态、空态
- 关键确认操作二次确认
- 错误页与恢复路径
