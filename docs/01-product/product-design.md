# 产品总设计

## 1. 产品定位

Digital Avatar 是一个面向个人专业用户与 AI-native 小团队的开源数字分身系统，目标是把用户的风格、知识、偏好和工作方式沉淀为一个可控、可审计、可持续成长的数字分身。

## 2. UVP

### 2.1 目标用户
- 个人专业用户：开发者、产品经理、研究者、内容创作者、重度 AI 用户
- AI-native 小团队：创业团队、顾问团队、教学与研究小组

### 2.2 价值主张
- 风格复刻：输出更像用户本人
- 可控执行：在授权范围内辅助完成真实任务
- 长期记忆：保留经过确认的高价值信息
- 审计可追溯：关键动作、策略命中与状态变化可回放
- 分身成长报告：把系统最近学到了什么、依据是什么、因此接下来更适合帮你做什么持续解释给用户看

### 2.3 当前不优先场景
- 企业级多租户与 IAM
- 插件市场与高自治代理平台
- 大规模面向 C 端聊天产品

## 3. 核心问题

1. 用户每次都要重复教 AI 自己的风格与偏好。
2. 历史上下文无法稳定继承。
3. AI 能生成内容，但执行边界不清晰。
4. 缺少长期记忆沉淀与纠错机制。
5. 缺少可信审计链路。
6. 冷启动门槛高，价值感知慢。

## 4. 设计原则

1. 先验证闭环，再扩展生态。
2. 先冷启动可体验，再增加复杂度。
3. 先安全可控，再提升自治。
4. 先本地优先，再考虑云端扩展。
5. 先冻结契约，再并行开发。
6. 默认可审计，默认最小权限。
7. 用户可确认、可纠正、可撤回、可删除、可导出。

## 5. MVP 闭环

1. 用户登录并创建 Avatar。
2. 导入最小资料并生成 Persona。
3. 创建默认 Agent。
4. 执行首个任务。
5. 捕捉候选记忆并由用户确认。
6. 写入长期记忆并保留审计链路。

## 6. 冷启动设计

- 提供 Demo Avatar
- 提供 Persona 模板：开发者 / 产品经理 / 创作者 / 分析师
- 引导式首启流程
- 样本导入最小化
- 推荐首个任务模板
- 提供 Avatar 成长报告

## 7. 分身成长报告（Avatar Growth Report）

### 7.1 功能定位
- 分身成长报告用于把“系统最近学到了什么、依据是什么、因此接下来能更好地帮你做什么”解释给用户看。
- 它是当前 Dashboard / Avatar 详情中的核心体验强化项，不是新的自治执行引擎。
- 它要把“越用越懂你”的价值显性化，同时保持可信、克制、可追溯。

### 7.2 允许使用的数据来源
- 当前激活 Persona：代表当前稳定画像。
- `confirmed` Memory：代表已经被用户确认的长期信息。
- 近期 Task：代表最近协作行为与使用活跃度。
- Audit：代表成长结论背后的证据链与来源入口。

### 7.3 明确不允许的行为
- 不允许把 `pending_confirm` 或 `rejected` 记忆写成“系统已学会”的结论。
- 不允许静默改写 Persona。
- 不允许把它写成开放插件、多智能体市场或高自治平台能力。
- Demo Avatar 如展示成长报告，必须明确为示例/seeded data。

## 8. 成功标准

- 首次可用时间 ≤ 15 分钟
- 冷启动完成率 ≥ 60%
- 记忆确认率 ≥ 30%
- 风格匹配感知 ≥ 7/10
- 任务完成率 ≥ 80%
- 高风险阻断准确率 ≥ 95%

## 9. 产品边界

### MVP 必须完成
- 登录认证
- Avatar / Persona / Agent / Task / Memory / Audit 主链路
- 基础 Policy 检查
- Demo 与新手引导
- Web 控制台
- Docker 一键部署

### MVP 明确不做
- 任意插件动态执行
- 高自治多代理编排
- 企业级多租户权限体系
- 桌面端正式交付
